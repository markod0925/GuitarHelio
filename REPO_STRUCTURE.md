# Repository Structure Guide

This document is a quick repository map to:
- find files faster;
- understand where core responsibilities live;
- reduce onboarding time when modifying a feature.

## 1) Top-level map

```text
.
|-- src/                    # TypeScript application code (core web game)
|-- test/                   # Vitest unit tests
|-- public/                 # static assets served by Vite (songs, UI images)
|-- assets/                 # web runtime assets/models (for example basic-pitch model)
|-- tools/                  # versioned local tools (audio->midi converter wrapper)
|-- scripts/                # CLI operational/utility scripts
|-- android/                # Capacitor/Android project + native plugin bridge
|-- third_party/            # vendored native dependencies (neuralnote/tempocnn/onnxruntime)
|-- samples/                # sample input/output files for manual tests
|-- dist/                   # Vite build output (generated)
|-- node_modules/           # npm dependencies (generated)
|-- README.md               # operational usage guide
|-- GDD.md                  # product/gameplay specifications
|-- package.json            # npm scripts and dependencies
|-- vite.config.ts          # bundling/dev server configuration
|-- capacitor.config.ts     # Capacitor configuration
```

## 2) Main entry points

- `src/app/main.ts`: Phaser bootstrap + scene registration (`BootScene`, `SongSelectScene`, `PlayScene`).
- `src/ui/BootScene.ts`: global asset preload and transition to the start screen.
- `src/ui/SongSelectScene.ts`: song catalog, settings/tuner overlays, MIDI/MP3/OGG import.
- `src/ui/PlayScene.ts`: gameplay loop, scoring, playback, runtime controllers.
- `package.json`: official commands (`dev`, `build`, `test`, `cap:*`, `android:*`, converter build).

## 3) `src/` structure by domain

- `src/app/`: app bootstrap, global config, settings/session persistence.
- `src/audio/`: mic input, pitch detector, scheduler, synth, and playback utilities.
- `src/game/`: runtime state logic and scoring.
- `src/guitar/`: tuning, fingering mapping, target-note generation.
- `src/midi/`: MIDI parsing/loading and tempo map.
- `src/platform/`: platform-specific adapters (web/native converter and song catalog/import).
- `src/types/`: shared types and custom declarations.
- `src/ui/`: Phaser scenes/rendering + HUD/overlays.
- `src/ui/play/controllers/`: specialized PlayScene controllers (lifecycle, playback, debug, layout, UI).

## 4) Tests, tooling, and pipeline

- `test/*.test.ts`: unit suite for core modules (tempo map, scoring, state machine, playback, converter mode).
- `tools/audio-midi-converter/`: Node wrapper for audio->MIDI conversion and native CLI build.
- `scripts/*.mjs`: operational utilities (startup benchmark, sample midi, CLI player, midi/audio helpers).
- `third_party/neuralnote_core`: C++ note transcription engine.
- `third_party/tempocnn_core`: C++ tempo estimation engine.
- `third_party/onnxruntime/*`: ONNX runtime for target platforms.

## 5) Assets and content

- `public/songs/manifest.json`: song catalog used by the Start Screen.
- `public/songs/<song-id>/`: song assets (`song.mid`, optional `song.mp3`, cover).
- `public/ui/`: UI images/placeholders/logo.
- `assets/models/basic-pitch/`: web-side basic-pitch model.
- `android/app/src/main/assets/`: assets packaged into Android.

## 6) Android/native area

- `android/app/src/main/java/.../NeuralNoteConverterPlugin.java`: native Capacitor plugin.
- `android/app/src/main/cpp/neuralnote_converter_jni.cpp`: JNI bridge to native converters.
- `android/app/src/main/cpp/CMakeLists.txt`: native library build config.
- `android/app/src/main/assets/neuralnote-model/`: NeuralNote models.
- `android/app/src/main/assets/tempo-model/fcn.onnx`: tempo model.

Note: `android/app/.cxx/` and `android/app/build/` contain artifacts generated by native builds.

## 7) Guide documents in the repo

- `README.md`: setup, runbook, QA, and main workflows.
- `GDD.md`: functional reference (project rule: update when specifications change).
- `IMPLEMENTATION_PLAN.md`: implementation plan.
- `IMPLEMENTATION_QA_CHECKLIST.md`: compliance checklist.
- `MANUAL_QA_RUNTIME_SUITE.md`: desktop/android runtime manual tests.
- `AGENTS.md`: operational instructions for agent/tooling.

## 8) Quick search (useful commands)

```bash
# list useful tracked files (excluding dependencies/build output)
rg --files -g "!node_modules" -g "!dist" -g "!android/app/.cxx" -g "!android/app/build"

# search where a class/method is used
rg "PlayScene|SongSelectScene|PitchDetectorService" src test

# search audio->MIDI conversion endpoints/imports
rg "convert|neuralnote|tempo" src tools scripts android

# search local persistence points (settings/highscore)
rg "localStorage|session|highScore|save" src

# search Android native/plugin wiring
rg "NeuralNoteConverter|jni|CMake|capacitor" android src/platform
```

## 9) Practical navigation conventions

- Gameplay changes: start from `src/ui/PlayScene.ts`, then `src/game/*`, `src/guitar/*`, `src/audio/*`.
- Start Screen/import changes: start from `src/ui/SongSelectScene.ts` and `src/platform/*`.
- Conversion pipeline changes: start from `tools/audio-midi-converter/*` and `third_party/*`.
- Mobile packaging changes: start from `capacitor.config.ts` and `android/*`.
- Song catalog changes: start from `public/songs/manifest.json`.

## 10) What is generated (not source)

- `node_modules/`
- `dist/`
- `android/app/.cxx/`
- `android/app/build/`
- `android/app/src/main/assets/public/assets/*` (web bundle synced by Capacitor)

Keeping "source code" and "build artifacts" mentally separate helps avoid editing in the wrong location.
